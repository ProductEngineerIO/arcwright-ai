## What Could Be Improved

**Aspirational `__all__` exports (Story 1.1 — HIGH finding).** Six package `__init__.py` files shipped with exports that didn't exist yet. Root cause: story spec listed planned exports as if real, and the implementation agent took them literally. Fix going forward: placeholder modules ship with empty `__all__` only; no aspirational exports.

**RUF022 (`__all__` alphabetical ordering) repeated as warning rather than absorbed as habit.** Issue first caught in Story 1.2. Then added as a "watch out" note in 1.3, 1.4, and 1.5 dev notes. The lesson was caught but not internalized. Symptom of no persistent agent memory — each story starts fresh.

**Missing `__init__.py` in test subdirectories (Story 1.1 — HIGH finding).** Nine test subdirectories created without `__init__.py`. Caught in review, but should have been in acceptance criteria from the start.

**Pydantic v2 mutable default fields (Story 1.2 — HIGH finding).** `ProvenanceEntry` shipped with `alternatives: list[str] = []` without `Field(default_factory=list)`. Classic Pydantic v2 pitfall. Needs to be on the standard code review checklist, not just story dev notes.

**Root cause analysis of recurring patterns.** The underlying issue: in a system with no persistent agent memory, the story spec *is* the memory. Lessons learned during Epic 1 were being added to dev notes *reactively* (after review) rather than *proactively* (before dev). The fix is front-loading known gotchas into every story dev notes during story creation.

---

## Key Insights

1. **The story spec is the agent's memory.** Everything the agent needs to know during implementation must be in the dev notes before it starts. This is the only continuity mechanism across stateless agent invocations.

2. **Pattern prevention beats pattern detection.** AI review is good at catching patterns. But the goal is preventing known patterns from occurring at all. Convert caught-in-review lessons into pre-armed defenses in story specs.

3. **Architecture fidelity in Epic 1 pays forward into every subsequent epic.** `core/` being clean and correct means Epics 2–7 build on a stable foundation. The investment in getting it right here was worth it.
