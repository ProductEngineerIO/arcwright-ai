# Sprint Change Proposal — BMAD 6.1 Framework Upgrade

**Date:** 2026-03-15
**Triggered by:** BMAD Method v6.1.0 release (2026-03-12)
**Change scope:** Minor — Development infrastructure only
**Approved by:** Ed

---

## 1. Issue Summary

BMAD Method has released v6.1.0, the biggest architectural overhaul since v6. The project is currently on **v6.0.3** (installed 2026-02-26). The 6.1 release converts all workflows, agents, and tasks to a unified **skills-based architecture** with `SKILL.md` entrypoints, removes the legacy `workflow.xml` engine, introduces Edge Case Hunter for code review, fixes critical bugs (infinite code review loops, lost brainstorming ideas), and reduces package size by 91%.

This change was identified proactively — the current installation is functional but will fall behind on bug fixes, new capabilities, and community alignment.

### Technical Impact — Source Code Coupling Audit

A full audit of arcwright-ai's Python source code confirmed **clean separation** between the product and BMAD framework internals:

- **Surface A (`_bmad/` framework):** Used by developers via IDE agents/workflows. Completely restructured by 6.1. Product code never reads these files.
- **Surface B (`_spec/` output artifacts):** Read by `context/injector.py`, `context/answerer.py`, `cli/status.py`, `cli/dispatch.py`. Format **unchanged** by 6.1.

**Conclusion:** Zero product code changes required. The upgrade is purely a development infrastructure operation.

## 3. Recommended Approach

**Direct Adjustment** — Add a new Epic 11 with 3 stories covering the safe upgrade lifecycle:

| Story | Points | Description |
|-------|--------|-------------|
| 11.1 Pre-Upgrade Audit & Backup | 3 | Audit `_bmad/` for custom modifications, create backup |
| 11.2 Execute BMAD 6.1 Installation | 5 | Run installer, verify framework migration |
| 11.3 Post-Upgrade Verification | 5 | Full test suite, dev workflow smoke test, cleanup |

**Total:** 13 points | **Risk:** Low | **Timeline impact:** None (independent of current work)

**Rationale:** The project's architecture was designed with clean separation between BMAD framework and product code (all `_spec/` access goes through configurable `MethodologyConfig.artifacts_path`). The 6.1 upgrade carries minimal technical risk but provides meaningful benefits — bug fixes, Edge Case Hunter for Phase 4 code review, and alignment with the active BMAD community.
